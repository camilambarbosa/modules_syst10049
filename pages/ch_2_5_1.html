Development in web environment

Publishing your website

It is a standard practice to development applications in a development environment, and then upload the application to a production environment. As changes and additions are made in the development environment, the production environment is synchronized accordingly.

Work with sftp and ssh protocols.

If you don't already have it installed on your machine, you can download FileZilla Client from

https://filezilla-project.org/download.php?type=client

Make sure the type is client for your machine.

Publishing using FileZilla:
STEP 1:
Start filezilla application. Log in with your credentials onto dev.fast.sheridanc.on.ca
STEP 2:
Navigate to the appropriate location on the server (right panel) /home/userid/public_html/syst10049
STEP 3:
Navigate to the appropriate location on the client (left panel) c:/public_html/syst10049
STEP 4:
Drag and drop the objects from one location (local machine) to the other (server machine).
Publishing a project using sftp:
  • STEP 1:  Start the psftp application. Log in with your credentials onto dev.fast.sheridanc.on.ca
  • STEP 2:  On server, position yourself in /home/userid/public_html:
    cd public_html/syst10049
  • STEP 3:  On your laptop, position yourself in c:/public_html directory
    lcd c:/public_html/syst10049
  • STEP 4:  Upload the entire project:
    mput -r projectName
Note: The steps are exactly the same, instead of giving commands, you use the graphical interface. Both tools use FTP protocol. (Make sure you always use the "secure" versions.
