Defining document structure

HTML Document

HTML Document is a text file containing hypertext markup language; interpreted and rendered on the screen by a browser. Various components and concepts are important to understand and keep in mind as you develop web applications. Both web sites and web applications use web pages or, more formally, HTML Documents.

Document Type

Every HTML document we create should declare its document type (doctype) in the first line. This will identify which specification of HTML is included so the browser knows how to interpret the markup. With HTML5, the general doctype declaration is
<!DOCTYPE html>
This is the very first line of code in any HTML page you create. Keep in mind that code examples you find online may contain older outdated or irrelevant approaches.

Document Structure

Each HTML document has a single root element, <html>, and is divided into two sections: <head> and <body>.

Head Section

The purpose of the <head> section is to describe the document itself. The children of head element include <meta> to describe the document, <link> to include stylesheet files and other assets, <styles> to embed styles, <script> to include scripts, and <title> in window tab.

Body Section

The purpose of the <body> section is to organize content for presentation by marking it up with HTML elements. The body element contains all marked up content that is presented to the user.
